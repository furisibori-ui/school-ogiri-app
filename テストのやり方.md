# 学校プロジェクト - テストのやり方（初めての人向け）

**このプロジェクト（school-ogiri-app）は「位置情報連動型・架空学校生成」だけ**です。  
3Dプレイ（WASD＋ジャンプ）は **別プロジェクト `play-3d-app`** にあります（同じ plan02 フォルダ内）。

---

## 1. 用意するもの

- **Node.js** が入っていること（入っていない場合は [Node.js の公式サイト](https://nodejs.org/) から「LTS」をダウンロードしてインストール）。**管理者権限がなくて公式インストールができない**場合は → `plan02/管理者権限なしでNodeを使う.md` を参照）
- **Cursor** または **VS Code** でこのフォルダ（`school-ogiri-app`）を開いていること

---

## 2. ターミナルを開く

1. Cursor のメニューで **「ターミナル」→「新しいターミナル」** を選ぶ（または **Control + `**）
2. 下や横に黒い画面（ターミナル）が出る
3. **今いる場所**がこのプロジェクトの中か確認する  
   - 表示が `.../plan02/school-ogiri-app` のように **school-ogiri-app** で終わっていれば OK  
   - 違う場合は、次のコマンドで移動する：
     ```bash
     cd /Users/d18212/Desktop/plan02/school-ogiri-app
     ```

---

## 3. パッケージを入れる（初回だけ）

ターミナルで、次のコマンドを打って Enter を押す。

```bash
npm install
```

- 初回は 1〜2 分かかることがある
- 「added xxx packages」のような表示が出れば成功

---

## 4. 開発サーバーを起動する

同じターミナルで、次を打って Enter：

```bash
npm run dev
```

- 「Ready in ...」「Local: http://localhost:3000」のような表示が出たら **起動できている**
- **このターミナルは閉じない・止めない**（止めるとサーバーが止まる）

---

## 5. ブラウザで開いてテストする

1. **ブラウザ**（Chrome や Safari など）を開く
2. アドレス欄に **`http://localhost:3000`** を入力して Enter

### このプロジェクトでできること

- 地図をクリックして場所を選ぶ
- 「テスト生成（東京タワー周辺）」で架空の学校サイトを生成

---

## 6. 終わるとき

- ターミナルで **Ctrl + C**（Mac は **⌃ + C**）を押すと、開発サーバーが止まる
- もう一度テストしたくなったら、**4 の「npm run dev」からやり直す**

---

## 3Dプレイ（WASD＋ジャンプ）を試したいとき

**別プロジェクト**で動かします。

1. フォルダ **`plan02/play-3d-app`** を Cursor で開く
2. その中の **「テストのやり方.md」** を見て、同じようにターミナルで `npm install` → `npm run dev` を実行
3. ブラウザで `http://localhost:3000` を開く（3D の方はこのプロジェクトのサーバーを止めてから、play-3d-app で起動した場合）

---

## 7. GitHub に上げたいとき（任意）

### このフォルダを Git の管理下にする（まだの場合）

ターミナルで、**school-ogiri-app のフォルダにいる状態**で：

```bash
git init
```

### リポジトリを作ってプッシュする

1. [GitHub](https://github.com) にログインして、**「New repository」** で新しいリポジトリを作る（名前は例：`school-ogiri-app`）
2. 作った画面に「…or push an existing repository from the command line」と出ているので、そこに書いてあるコマンドをターミナルで実行する。だいたい次のような形です（URL は自分のリポジトリのものに置き換える）：

   ```bash
   git remote add origin https://github.com/あなたのユーザー名/school-ogiri-app.git
   git branch -M main
   git add .
   git commit -m "初回コミット"
   git push -u origin main
   ```

3. 初回の `git push` で GitHub のログインを求められたら、ブラウザで認証する

---

## 困ったとき

| 症状 | やること |
|------|----------|
| `npm: command not found` | Node.js をインストールする（上記 1）。管理者権限がなくて入れられない場合は `plan02/管理者権限なしでNodeを使う.md` を参照 |
| `npm install` でエラー | ターミナルで `cd` が `school-ogiri-app` になっているか確認する |
| ブラウザで「接続できません」 | `npm run dev` を実行したターミナルが動いたままか確認する |

まだ分からないところがあれば、「何をした・何が表示された」を教えてもらえれば、次の一手を具体的に書きます。
